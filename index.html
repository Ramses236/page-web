Aprendiendo Programación Web paso a paso

Esta página ha sido desarrollada como parte de un proceso de aprendizaje en programación web. Su objetivo es aplicar de manera progresiva los conceptos fundamentales de HTML, CSS y JavaScript, mediante una estructura clara, laboratorios interactivos y ejemplos prácticos que facilitan la comprensión del funcionamiento de una aplicación web.

HTML

Estructura y contenido de la web.

Historia sobre HTML

HTML nació para estructurar documentos en la web. Hoy se usa para crear páginas accesibles y semánticas usando etiquetas como header, main, section y footer.

Nociones importantes

  • Semántica (header/main/section/footer)
  • Atributos: id, class, src, href
  • Formularios: label + input

Ejemplo

<section>
    <h1>Hola</h1>
    <p>Mi primera página</p>
    </section>

Mini ejercicio: crea un formulario con nombre y email.

CSS

Diseño, layout y estilo visual.

Historia sobre CSS

CSS separa el contenido del diseño. Permite controlar colores, tipografías, layout (Flexbox/Grid) y adaptabilidad (responsive).

Nociones importantes

  • Box model y box-sizing
  • Flexbox y Grid
  • Pseudoclases: :hover, :focus

Ejemplo

.card{
    border-radius: 14px;
    padding: 14px;
    box-shadow: 0 10px 25px rgba(0,0,0,0.08);
    }

Mini ejercicio: haz un botón con hover y transición.

JavaScript

Interacción y lógica en el navegador.

Historia sobre JavaScript

JavaScript permite manipular el DOM, responder a eventos y crear experiencias dinámicas (formularios, animaciones, apps).

Nociones importantes

  • Variables y funciones
  • DOM: querySelector
  • Eventos: click, input

Ejemplo

document.querySelector("button").addEventListener("click", () => {
    alert("Hola!");
    });

Mini ejercicio: al hacer clic cambia el texto de un párrafo.

Nociones importantes

Chuleta rápida para no olvidar lo esencial.

HTML

Semántica Atributos Formularios Accesibilidad SEO básico
  • Semántica: usa header, main, section, article, footer.
  • Atributos clave: id, class, href, src, alt, title, data-*.
  • Formularios: siempre <label for=""> + id en el input.
  • Imágenes: alt descriptivo; loading="lazy" en imágenes no críticas.
  • Accesibilidad: botones de verdad (<button>), roles solo si es necesario, buen contraste.

CSS

Cascada Especificidad Flex Grid Responsive Unidades
  • Cascada: último gana (si misma especificidad). Evita guerras de CSS.
  • Especificidad: id > class > etiqueta. Evita abusar de !important.
  • Box model: usa box-sizing: border-box para cálculos predecibles.
  • Layout: Flex = 1D (fila/columna), Grid = 2D (filas+columnas).
  • Responsive: usa max-width, clamp() y media queries cuando toque.
  • Unidades: rem para tipografía, %/fr para layouts, px para bordes finos.
  • Performance: transiciones en transform y opacity (mejor que top/left).

JavaScript

DOM Eventos Async Fetch Storage
  • DOM: querySelector, createElement, classList, dataset.
  • Eventos: usa delegación cuando hay muchos elementos (container.addEventListener).
  • Async: aprende Promise, async/await y manejo de errores (try/catch).
  • Fetch: verifica res.ok, parsea JSON con cuidado, maneja CORS.
  • localStorage: guarda strings; para objetos usa JSON.stringify/JSON.parse.
  • Buenas prácticas: funciones pequeñas, nombres claros, no repetir lógica (DRY).

Checklist rápido

  • ¿Tus anchors apuntan a id existentes?
  • ¿Inputs con label?
  • ¿Contraste OK y navegación con teclado?
  • ¿CSS sin !important innecesario?
  • ¿JS sin errores en consola?
